TASK 3: Deploy to Kubernetes
=============================

PREREQUISITES:
  - Enable Kubernetes in Docker Desktop (Settings > Kubernetes > Enable Kubernetes)
  - OR use Minikube: minikube start
  - Verify: kubectl get nodes

STEP 1: Create namespace and deploy
  kubectl apply -k k8s/

STEP 2: Verify deployment
  kubectl get all -n mlops-assignment
  kubectl get pods -n mlops-assignment
  kubectl get service heart-disease-api-service -n mlops-assignment

STEP 3: Wait for pod to be Running and Ready
  kubectl wait --for=condition=Ready pod -l app=heart-disease-api -n mlops-assignment --timeout=300s

STEP 4: Access the API
  
  Docker Desktop Kubernetes (LoadBalancer on localhost):
    Health endpoint:
      curl http://localhost/health
    
    Prediction endpoint:
      curl -X POST http://localhost/predict \
        -H "Content-Type: application/json" \
        -d @samples/predict_sample.json
  
  Minikube (use tunnel or service URL):
    minikube tunnel
    
    Then use:
      SERVICE_IP=$(kubectl get service heart-disease-api-service -n mlops-assignment -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
      curl http://$SERVICE_IP/health

STEP 5: View logs
  kubectl logs -f deployment/heart-disease-api -n mlops-assignment

STEP 6: Clean up
  kubectl delete -k k8s/
